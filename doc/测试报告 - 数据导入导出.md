### 问题 5: Excel 数字类型单元格处理（已修复）

**现象**: 
- 考勤/薪资 Excel 导入时报 `IllegalArgumentException: For input string: "3.0"`

**原因**: 
- Excel 中数字类型的单元格被读取为 double 类型（如 `3.0`）
- 原代码强制转换为字符串类型失败

**修复方案**:

**修改文件**: [`ImportExportService.java`](file:///home/alex/hr-system/src/main/java/com/example/hr/service/ImportExportService.java#L321-L340)

```java
private String getString(Row r, int idx) {
  Cell cell = r.getCell(idx);
  if (cell == null) return "";
  
  // 根据单元格类型分别处理
  switch (cell.getCellType()) {
    case STRING:
      return cell.getStringCellValue().trim();
    case NUMERIC:
      double value = cell.getNumericCellValue();
      if (value == (long) value) {
        return String.valueOf((long) value);  // 3.0 → "3"
      }
      return String.valueOf(value);  // 3.14 → "3.14"
    case BOOLEAN:
      return String.valueOf(cell.getBooleanCellValue());
    default:
      return "";
  }
}
```

**效果**: 
- ✅ 支持字符串单元格
- ✅ 支持数字单元格（整数自动去掉小数部分）
- ✅ 支持布尔单元格

## 📝 CSV 格式要求

### 员工导入格式
```csv
employeeNo,name,department,title,phone,email,hireDate,status

```

**必填字段**: 8 列

- department: 部门
- title: 职位
- phone: 电话
- email: 邮箱

- status: 状态

---
