# 前端数据渲染问题总结

## 更新时间

2026-05-04

## 问题现象

权限管理 tab 反复出现数据无法渲染的情况，修复后仍会在不同场景下复现。

## 根因分析

### 数据加载与渲染的时序耦合过于脆弱

权限 tab 的初始化被拆成三个独立调用，分布在 `ensureTabData` 和 `initTabEvents` 中：

- `loadRoles` / `applyRoles` — 加载或渲染角色列表
- `loadPermissions` — 加载并渲染权限表格
- `initPermissionTab` — 绑定按钮事件

外部调用者负责拼凑这些函数，但没有保证执行顺序和状态完整性。

### 依赖链断裂

`loadPermissions` 内部依赖 `selectedRole` 变量来决定渲染哪个角色的权限：

```javascript
async function loadPermissions() {
  const list = await apiRequest(API.permissions.list);
  initCache.permissions = list;
  if (selectedRole) {
    const perms = list.filter(p => p.role === selectedRole);
    renderPermissions(perms);
  }
}
```

但 `selectedRole` 只在 `selectRole` 函数中设置。首次进入 tab 时，`selectedRole` 为 null，权限表格不会渲染。

### 缓存分支导致状态不完整

`ensureTabData` 中存在两条路径：

```javascript
if (initCache.roles) {
  applyRoles(initCache.roles);  // 只渲染列表，不设置 selectedRole
} else {
  await safeLoad("roles", loadRoles);  // loadRoles 内部会 autoSelectFirst
}
```

当角色数据已有缓存时，走 `applyRoles` 分支，跳过了自动选中逻辑。此时 `selectedRole` 仍为 null，后续 `loadPermissions` 直接跳过渲染。

### 职责不清晰

`loadRoles` 同时承担了四个职责：数据获取、缓存写入、DOM 渲染、自动选中。缓存命中时只执行了其中一部分（DOM 渲染），状态就不完整。

## 修复方案

提取 `autoSelectFirst` 为独立函数，在缓存命中时也调用，确保首次进入权限 tab 时自动选中第一个角色：

```javascript
function autoSelectFirst(roles) {
  if (!selectedRole && roles && roles.length > 0) {
    selectRole(roles[0].role, roles[0].roleMode);
  }
}
```

在 `ensureTabData` 的缓存分支中补充调用：

```javascript
if (initCache.roles) {
  applyRoles(initCache.roles);
  autoSelectFirst(initCache.roles);
}
```

## 根本解法

应该有一个统一的初始化入口函数，内部按正确顺序执行：

```javascript
async function initPermissionsTab() {
  await loadRoles();       // 获取角色 + 缓存 + 渲染 + 自动选中
  await loadPermissions(); // 获取权限 + 渲染当前角色
  bindPermissionEvents();  // 绑定按钮事件
}
```

外部只需调用这一个函数，不需要关心内部依赖关系。
