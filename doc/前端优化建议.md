# 前端代码审查与优化建议

## 审查范围

Vue3 迁移后的全部前端代码：App.vue、7 个视图组件、2 个通用组件、7 个 Pinia Store、7 个 API 模块、axios 拦截器、Vite 构建配置。

## 一、架构级问题

### 1. 所有标签页组件同时挂载，API 请求全量并行

**现状：** App.vue 中 7 个标签页全部渲染在 DOM 中，CSS `display: none` 隐藏非当前标签。Vue 的 onMounted 在组件挂载时全部触发。登录后一瞬间 7 个模块同时发起 API 请求，即使用户只看仪表盘。

```
<div class="tab-content" :class="{ active: currentTab === 'dashboard' }">
  <DashboardView />   <!-- onMounted → fetchSummary() -->
</div>
<div class="tab-content" :class="{ active: currentTab === 'employees' }">
  <EmployeesView />   <!-- onMounted → fetchList() + init() -->
</div>
<!-- 共 7 个同时触发 -->
```

当前数据量小，影响不明显。但在生产环境中每个标签页可能拉取大量数据，7 个请求同时发出会浪费带宽和服务端资源。

**改进方案：** 用 `v-if` 或动态组件 `<component :is="...">` 实现懒加载。首次打开时才触发数据请求，后续切换从 Pinia store 缓存读取。

### 2. vite.config.js 代理端口是死配置

**现状：** `server.proxy` 配置指向 `localhost:8080`，但应用运行在 `18080` 端口。整个开发周期中从未使用 `vite dev`，始终通过 `npm run build` + `spring-boot:run` 部署。此配置既是死代码，也会误导后续开发者。

**建议：** 改为 18080，或添加注释说明仅用于前端独立开发场景。

### 3. window.Chart = Chart 是绕路实现

**现状：** 仪表盘图表逻辑中 `ensureChart` 检查 `window.Chart` 是否存在——这是遗留自 CDN 加载 Chart.js 的判断。迁移后使用 ESM 导入，`window.Chart` 始终为 undefined，所以加了 `window.Chart = Chart` 补丁。

```
import Chart from 'chart.js/auto'
window.Chart = Chart     // 补丁，为了让 ensureChart 的条件通过
```

**改进方案：** 去掉 `window.Chart` 检查和赋值。ESM 导入的 `Chart` 已经在模块作用域中可用，`ensureChart` 直接使用模块变量即可：

```
const Chart = ...  // 模块顶层
function ensureChart(id, config) {
  const canvas = $(id)
  if (!canvas) return null
  ...
}
```

### 4. ConfirmDialog 是死代码，与原生 confirm() 并存

**现状：** App.vue 中定义了 ConfirmDialog 组件及 `confirmDialog` 状态变量，但从未被赋值使用。所有删除确认全部采用原生 `confirm()`。同时 ConfirmDialog 使用了 `<style scoped>`，类名也不匹配 modal.css。

**建议：** 两条路径选一条——要么删除 ConfirmDialog 及其相关状态，统一用原生 `confirm()`；要么用 ConfirmDialog 复刻 modal.css 中的 `.confirm-modal` 类，替换所有原生 `confirm()`。

## 二、代码质量问题

### 5. 请求错误无统一处理

**现状：** axios 响应拦截器仅 `return Promise.reject(error)`，未提取业务错误信息。每个组件各自 `alert(err.message)`，但 axios 网络超时时 err.message 是 "Network Error"，业务 500 时 err 结构又不同。

**改进方案：** 在响应拦截器中统一处理：

```
apiClient.interceptors.response.use(
  (response) => response.data,
  (error) => {
    const msg = error.response?.data?.message
      || error.message
      || '请求失败'
    alert(msg)
    return Promise.reject(error)
  }
)
```

组件层不再需要单独 `alert(err.message)`，只需 `try/catch` 处理操作后的 UI 重置。

### 6. 仪表盘图表配置严重重复

**现状：** `renderCharts()` 中对每个图表，数据配置出现了两次——一次传给 `ensureChart`，一次传给 `updateChart`。例如员工规模趋势图，`labels: hc.map(i => i.month)` 写了两次、`datasets` 的 5 行配了两次。五个图表共约 10 份重复数据配置。

**改进方案：** 提取一个 `renderChart(id, labels, datasets, chartConfig)` 函数，创建和更新合二为一，`renderCharts()` 从约 50 行缩至约 25 行。

### 7. 所有 Store 操作后全量刷新列表

**现状：** create/update/delete 后全部重新 `fetchList()`。这是原始 JavaScript 的逻辑遗留——当时没有状态管理，只能重新拉数据刷新 table.innerHTML。现在有 Pinia 管理状态，理论上删除一条记录只需 `this.list.splice(index, 1)`，新增一条只需 `this.list.push(data)`。

**注意：** 后端某些操作（如入职/复职）会改变关联数据，全量刷新保证了数据一致性。是否需要优化取决于数据量和交互频率，当前数据量小，不构成性能瓶颈。此条目作为"可优化但不急"的建议记录。

### 8. 不可达代码

- `useEmployeeStore.employeeOptions` getter 定义了但从未被任何组件引用
- `App.vue` 中 `showRegister` ref 被"注册新用户"按钮绑定，但没有任何处理逻辑——`v-if="showRegister"` 的模板不存在
- `confirmDialog` reactive 对象定义了，ConfirmDialog 也渲染了，但 `onConfirm` 回调从未被赋值

## 三、体验问题

### 9. 缺少加载状态指示

**现状：** 每个 Store 都有 `loading` 状态，但所有组件模板中没有任何加载指示器——没有骨架屏、没有 spinner、没有"加载中..."文字。网络慢时用户面对空白页面。

**改进方案：** 在两个关键位置加入简单的加载提示：

表格区域：`<tr v-if="store.loading"><td colspan="..." class="muted">加载中...</td></tr>`

员工卡片区：`<div v-if="employeeStore.loading" class="subtitle">加载中...</div>`

### 10. 所有错误提示用原生 alert() 和 confirm()

**现状：** 系统已经有一套模态框样式体系（modal.css），但全部交互提示仍然使用原生浏览器弹窗。用户体验不统一。

**改进方案：** 如果保留 ConfirmDialog，统一用它替换所有 `confirm()` 调用。

### 11. 导入导出对 blob 响应的处理隐患

**现状：** `dashboard.js` API 文件中 `importExportApi.export()` 传递 `responseType: 'blob'`，而 axios 响应拦截器 `response => response.data` 会对所有响应解包。当前 blob 响应能正常工作是因为 axios 对 blob 类型的 `response.data` 本身就是 Blob 对象，但这是一个隐式依赖——如果未来修改拦截器逻辑，这个路径容易忽略测试。

**建议：** 对 blob 响应跳过拦截器处理：

```
apiClient.interceptors.response.use(
  (response) => response.config.responseType === 'blob' ? response : response.data,
  ...
)
```

## 四、改进优先级

| 优先级 | 内容 | 收益 | 工作量 |
|--------|------|------|--------|
| 高 | 标签页改为懒加载 | 登录后请求从 7+ 降到 1，首屏秒开 | 30 行改动 |
| 高 | 统一 axios 错误处理 | 所有错误提示可读可理解 | 10 行改动 |
| 中 | 删除死代码 | 消除混淆，减少维护负担 | 5 分钟内完成 |
| 中 | 精简图表配置 | renderCharts 从 50 行降到 25 行 | 10 分钟 |
| 低 | 添加 loading 指示 | 网络延迟时不再空白 | 每条 3 行 |
| 低 | 去掉 window.Chart 补丁 | 消除隐式全局变量 | 3 行改动 |
| 低 | 修正 vite 代理端口 | 配置与运行环境一致 | 1 行改动 |
