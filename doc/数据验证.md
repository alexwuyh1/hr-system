# 数据验证机制

> HR 系统所有数据验证规则汇总

---

## 验证架构

系统采用三层验证机制：

```
请求 → ① Bean Validation 注解 → ② 自定义验证器 → ③ Service 层业务校验
```

| 层级 | 职责 | 示例 |
|------|------|------|
| ① Bean Validation | 格式、非空、范围 | `@NotBlank`, `@Email`, `@PastOrPresent` |
| ② 自定义验证器 | 复杂格式、数据库查询 | `@Phone`, `@UniqueEmployeeNo` |
| ③ Service 层 | 业务逻辑、状态约束 | 离职状态检查、工号重复检查 |

---

## 一、Bean Validation 内置注解

### 员工请求 (EmployeeRequest)

| 字段 | 注解 | 规则 | 错误提示 |
|------|------|------|----------|
| employeeNo | `@NotBlank` | 不能为空 | 员工工号不能为空 |
| name | `@NotBlank` | 不能为空 | 员工姓名不能为空 |
| department | `@NotBlank` | 不能为空 | 部门不能为空 |
| title | `@NotBlank` | 不能为空 | 职位不能为空 |
| email | `@Email` | 邮箱格式 | 邮箱格式不正确 |
| hireDate | `@NotNull` + `@PastOrPresent` | 必填且不晚于今天 | 入职日期不能为空 / 不能晚于今天 |
| status | `@NotBlank` | 不能为空 | 员工状态不能为空 |
| phone | `@Phone` | 自定义（见下方） | — |

### 考勤请求 (AttendanceRequest)

| 字段 | 注解 | 规则 |
|------|------|------|
| employeeId | `@NotNull` | 必填 |
| workDate | `@NotNull` | 必填 |
| status | `@Pattern(Normal\|Late\|Absent\|Leave)` | 枚举值，默认 Normal |

### 薪资请求 (SalaryRequest)

| 字段 | 注解 | 规则 |
|------|------|------|
| employeeId | `@NotNull` | 必填 |
| salaryMonth | `@NotBlank` + `@Pattern(\d{4}-\d{2})` | YYYY-MM 格式 |
| baseSalary | `@NotNull` + `@PositiveOrZero` | ≥ 0 |
| bonus | `@NotNull` + `@PositiveOrZero` | ≥ 0 |
| deduction | `@NotNull` + `@PositiveOrZero` | ≥ 0 |

### 认证请求 (AuthRequests)

| 字段 | 注解 | 规则 |
|------|------|------|
| username | `@NotBlank` | 不能为空 |
| password | `@NotBlank` | 不能为空 |

## 三、Service 层业务校验

### 员工服务 (EmployeeService)

| 方法 | 校验规则 | 异常类型 |
|------|----------|----------|
| create() | 工号已存在且状态为"在职"时拒绝创建 | DuplicateResourceException |
| update() | 工号被其他员工占用时拒绝更新 | DuplicateResourceException |
| resign() | 仅"在职"员工可离职 | InvalidStateException |
| rehire() | 仅"离职"员工可复职 | InvalidStateException |
| apply() | departmentId/positionId/gradeId/managerId 必须存在 | ResourceNotFoundException / EmployeeNotFoundException |

### 关键代码

```java
// create — 工号重复检查
employeeRepository.findByEmployeeNo(request.employeeNo)
    .ifPresent(existing -> {
        if ("在职".equals(existing.getStatus())) {
            throw new DuplicateResourceException("员工", request.employeeNo);
        }
    });

// update — 工号被他人占用检查
employeeRepository.findByEmployeeNo(request.employeeNo)
    .ifPresent(existing -> {
        if (!existing.getId().equals(id)) {
            throw new DuplicateResourceException("员工工号", request.employeeNo);
        }
    });

// resign — 状态前置检查
if (!"在职".equals(employee.getStatus())) {
    throw new InvalidStateException("员工", employee.getStatus(), "在职");
}
```

## 四、UI 层输入提示

表单通过 `placeholder` 属性提供输入指引：

| 字段 | 提示文本 |
|------|----------|
| 工号 | 唯一标识，如 EMP001 |
| 姓名 | 员工真实姓名 |
| 邮箱 | example@company.com |
| 电话 | 11 位手机号，如 13800138000 |

## 五、验证异常处理

所有验证失败统一由全局异常处理器捕获，返回 JSON 格式错误：

```json
{
  "code": "VALIDATION_ERROR",
  "message": "验证失败：具体错误信息",
  "timestamp": "2026-05-02T11:22:28.297430671",
  "path": "/api/employees"
}
```
