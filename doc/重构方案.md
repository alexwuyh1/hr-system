## 3. 三表合一方案

### 3.1 现有三表结构

部门表包含标识、名称、父部门字段，支持自引用树形结构。岗位表包含标识和名称字段，名称唯一。职级表包含标识、名称和等级数值字段，名称唯一。员工表通过三个外键分别关联这三张表。

### 3.2 新组织表设计

新建组织表替代上述三表，采用类型字段区分组织单元类别。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | 主键 | 组织单元唯一标识 |
| name | TEXT | 非空 | 组织单元名称 |
| type | TEXT | 非空 | 类型（部门、岗位、职级） |
| parent_id | INTEGER | 外键 | 关联父组织（自引用，仅部门使用） |
| level | INTEGER | 可空 | 等级数值（仅职级使用） |

类型字段枚举值为部门、岗位、职级三种。部门类型使用 parent_id 字段实现树形结构，岗位类型不使用 parent_id，职级类型使用 level 字段存储等级数值。

### 3.3 员工表调整

员工表中的三个外键字段合并为一个外键：

- 删除 `department_id`、`position_id`、`grade_id` 三个字段
- 新增 `org_id` 字段，关联组织表

### 3.4 组织表实体代码

```java
@Entity
@Table(name = "organizations")
public class Organization {
  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

  @Column(nullable = false)
  private String name;

  @Column(nullable = false)
  private String type;

  @ManyToOne(fetch = FetchType.EAGER)
  @JoinColumn(name = "parent_id")
  private Organization parent;

  @Column
  private Integer level;
}
```

## 6. 实施步骤

第一步，删除请假和加班模块的全部后端文件，包括模型、仓库、服务、控制器和数据传输对象。清理 AttendanceRuleEngineService、DashboardService、DataInitializer 中对请假加班的引用。

第二步，创建组织实体类和对应的数据访问层、服务层、控制器层。组织表通过类型字段区分部门、岗位、职级，通过 parent_id 实现部门树形结构，通过 level 字段存储职级等级。

第三步，修改员工实体类，将三个外键字段替换为单一 org_id 外键。更新员工相关的创建、查询、更新逻辑。

第四步，更新前端组织管理模块，适配新的组织表数据结构。删除考勤模块中请假和加班相关的前端代码和 UI 元素。

第五步，编写数据库迁移脚本，将现有部门、岗位、职级数据迁移到组织表，更新员工表的外键引用。
