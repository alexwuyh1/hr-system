## 详细修复

### 1. 更新员工返回 400 — 工号唯一性验证冲突

**原因：** `@UniqueEmployeeNo` 注解验证器在更新时无法排除当前员工自身，导致用自己的工号更新自己时报"工号已存在"。

**修复：**
- 从 [EmployeeRequest](file:///home/alex/hr-system/src/main/java/com/example/hr/dto/EmployeeRequest.java) 移除 `@UniqueEmployeeNo` 注解
- 在 [EmployeeService.update()](file:///home/alex/hr-system/src/main/java/com/example/hr/service/EmployeeService.java#L62-L75) 中手动检查工号是否被**其他**员工占用
- 保留 `@UniqueEmployeeNo` 注解定义供后续需要时使用

```java
// EmployeeService.update()
employeeRepository.findByEmployeeNo(request.employeeNo)
    .ifPresent(existing -> {
        if (!existing.getId().equals(id)) {
            throw new DuplicateResourceException("员工工号", request.employeeNo);
        }
    });
```

### 2. 仪表盘/考勤列表返回 500 — 孤立关联记录

**原因：** 测试删除员工后，其关联的考勤和薪资记录仍存在。由于 Attendance 和 Salary 使用 EAGER 加载 Employee，查询时触发 `JpaObjectRetrievalFailureException`。

**修复：**
- 清理数据库中已有的孤立记录（attendance id=2,3；salaries id=3）
- 测试脚本改为先删除关联数据再删除员工
- 测试脚本新增考勤/薪资关联数据清理逻辑

```bash
# 删除员工前先清理关联数据
ATT_LIST=$(curl -s "$BASE_URL/api/attendance" -H "$AUTH_HEADER")
echo "$ATT_LIST" | grep -o "\"id\":[0-9]*" | while read line; do
  ATT_ID_CLEAN=$(echo "$line" | cut -d: -f2)
  curl -s -X DELETE "$BASE_URL/api/attendance/$ATT_ID_CLEAN" -H "$AUTH_HEADER" > /dev/null
done
```

### 3. 创建考勤记录返回 400 — status 字段必填

**原因：** [AttendanceRequest](file:///home/alex/hr-system/src/main/java/com/example/hr/dto/AttendanceRequest.java) 的 `status` 字段标记了 `@NotBlank`，但创建考勤时未传该字段。

**修复：** 移除 `@NotBlank`，设置默认值 `"Normal"`

```java
@Pattern(regexp = "Normal|Late|Absent|Leave", message = "Invalid attendance status")
public String status = "Normal";
```

### 4. 薪资管理返回 403 — 测试脚本字段名不匹配

**原因：** 测试脚本使用了错误的字段名和 API 路径：
- `yearMonth` → 应为 `salaryMonth`
- `deductions` → 应为 `deduction`
- `/api/salary` → 应为 `/api/salaries`

**修复：** 更正测试脚本中的字段名和路径，与 [SalaryRequest](file:///home/alex/hr-system/src/main/java/com/example/hr/dto/SalaryRequest.java) 和 [SalaryController](file:///home/alex/hr-system/src/main/java/com/example/hr/controller/SalaryController.java) 保持一致。

### 5. 注册新用户返回 400 — 用户名重复

**原因：** 测试脚本固定使用 `testuser` 作为注册用户名，多次运行后该用户名已存在。

**修复：** 使用带时间戳的唯一用户名

```bash
TEST_USER="testuser_$(date +%s)"
```
