## 二、待修复问题

### 问题 1：JWT Secret 使用默认值

**配置位置：** [application.yml](file:///home/alex/hr-system/src/main/resources/application.yml#L44-L47)

```yaml
app:
  auth:
    jwt-secret: "change-this-secret-to-a-long-random-string"
    jwt-expire-minutes: 120
```

**风险：**
- 生产环境可被轻易伪造 token
- 任何知道默认 secret 的人都可以生成有效 token

**建议：**
- 使用环境变量：`${JWT_SECRET}`
- 生成强随机密钥：`openssl rand -base64 64`

---

### 问题 2：Token 中缺少用户角色信息

**当前 Token 结构：**
```java
// TokenService.java:37-44
return Jwts.builder()
    .setSubject(String.valueOf(userId))  // 仅包含 userId
    .setIssuedAt(Date.from(now))
    .setExpiration(Date.from(exp))
    .signWith(key, SignatureAlgorithm.HS256)
    .compact();
```

**问题：**
- 每次请求都需要查询数据库获取用户角色
- 增加数据库压力
- 响应延迟增加

**建议：**
- 在 JWT 中添加 `role` claim
- 验证时从 token 中直接读取角色

---

### 问题 3：权限检查无缓存

**代码位置：** [JwtAuthenticationFilter.java](file:///home/alex/hr-system/src/main/java/com/example/hr/config/JwtAuthenticationFilter.java#L94)

```java
if (!permissionService.isAllowed(user.getRole(), request.getMethod(), path)) {
```

**问题：**
- 权限检查依赖数据库查询
- 没有缓存机制
- 高并发性能差

**建议：**
- 添加权限缓存（如 Caffeine）
- 设置合理的过期时间

## 五、前端认证问题

### Token 存储
```javascript
// app.js:123-129
function persistToken(token) {
    authToken = token;
    if (token) {
        localStorage.setItem("hr_token", token);
    } else {
        localStorage.removeItem("hr_token");
    }
}
```

### 问题
1. **Token 过期处理**：前端没有处理 401 响应，不会自动跳转登录
2. **Token 刷新**：没有 refresh token 机制
3. **登录状态检查**：页面加载时不检查 token 有效性
