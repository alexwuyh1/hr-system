# UI 优化方案

## 更新说明（2026-05-01）
将平铺表单改为弹窗交互，移除考勤设置页，整合规则设置至考勤中心。员工管理页仅保留表格展示，新增员工功能移至员工中心。人脸签到/签退/验证移至考勤中心。

---

## 当前问题分析

### 1. 表单平铺问题
- 员工管理、考勤、薪资等模块的表单全部平铺在页面上
- 表单占用大量空间，用户需要滚动才能看到表格数据

### 2. 按钮事件无约束
- 点击按钮直接提交，没有模态窗口引导
- 缺少编辑功能，只有删除和状态切换

### 3. 交互流程不清晰
- 新增、编辑、删除操作没有统一的弹窗规范
- 验证错误使用 `alert()` 打断用户体验

---

## 优化方案

### 核心设计原则
1. **表格为主，弹窗为辅**：页面只展示数据表格和操作按钮
2. **模态窗口约束输入**：所有表单操作通过弹窗完成
3. **统一交互模式**：新增、编辑、删除都使用一致的弹窗风格

## 技术实现

### 1. Modal 组件 (`core/modal.js`)

#### 核心 API
```javascript
openModal(title, contentHTML, onSubmit, options)  // 打开表单弹窗
closeModal()                                       // 关闭弹窗
showConfirm(message, onConfirm)                    // 显示确认对话框
```

#### 功能特性
- 遮罩层点击关闭 / ESC 键关闭
- 表单验证拦截
- 打开/关闭动画
- 支持自定义按钮文本

### 2. 样式 (`styles/modal.css`)
- 遮罩层：半透明黑色背景，淡入动画
- 弹窗容器：白色背景，圆角，居中
- 表单布局：响应式，支持 `.form-grid-2` / `.form-grid-3`
- 移动端适配

## 交互流程图

```
用户操作
  │
  ├─ 新增 → 点击按钮 → 弹出空表单 → 填写 → 提交 → 刷新表格
  │
  ├─ 编辑 → 点击编辑 → 弹出预填表单 → 修改 → 提交 → 刷新表格
  │
  ├─ 删除 → 点击删除 → 弹出确认 → 确认 → 删除 → 刷新表格
  │
  ├─ 文件操作 → 点击按钮 → 弹出文件选择 → 确认 → 执行 → 刷新
  │
  └─ 取消 → 点击取消/遮罩/ESC → 关闭弹窗 → 无操作
```

## 代码示例

### Modal 组件使用示例

```javascript
// 新增员工
function openCreateEmployeeModal() {
  openModal(
    '新增员工',
    getEmployeeFormHTML(),
    async (formData) => {
      await apiRequest(API.employees.create, {
        method: 'POST',
        body: JSON.stringify(formData)
      });
      await loadEmployees();
    },
    { submitText: '入职' }
  );
}

// 删除确认
function confirmDelete(id) {
  showConfirm('确定要删除该员工吗？', async () => {
    await apiRequest(API.employees.delete(id), { method: 'DELETE' });
    await loadEmployees();
  });
}
```
