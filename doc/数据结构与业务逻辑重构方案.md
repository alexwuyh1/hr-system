# 数据结构与业务逻辑重构方案

更新时间：2026-05-04

## 一、重构目标

本次重构解决以下问题：岗位关联数据缺少业务层约束、手动打卡日期时间需自动获取、考勤规则计算功能冗余、员工表单部门职级需从岗位自动推断。

## 三、实施记录

### 3.1 岗位关联非空验证

最初尝试在 Organization 模型的 `@JoinColumn` 上设置 `nullable = false`，但发现部门和职级本身允许 parent/grade 为空，数据库级约束会影响所有组织类型。改为在 OrganizationController 的 `applyRelations` 方法中进行业务层验证：岗位类型必须提供 parentId 和 gradeId，否则抛出 IllegalArgumentException。

### 3.2 手动打卡自动获取时间

AttendanceRequest 删除 workDate、checkIn、checkOut 字段，只保留 employeeId 和 note。AttendanceService 的 apply 方法使用 LocalDate.now() 和 LocalTime.now() 自动填充。前端 attendance.js 的打卡表单只保留员工选择和备注输入。

### 3.3 删除考勤规则计算功能

删除 AttendanceRuleEngineService.java 文件。AttendanceRuleController 只保留 getRule 和 update 两个端点。api.js 删除 calculate 和 calculateRange 路由。attendance.js 删除规则弹窗中的计算日期输入字段和 initAttendanceRules 函数。

### 3.4 员工表单部门职级自动推断

EmployeeService 的 apply 方法删除 orgId 独立设置逻辑，改为从 positionRef.getParent() 自动推断 orgRef。前端 employees.js 删除部门输入框和职级输入框，删除表单提交时的 orgId 推断逻辑。

### 3.5 数据初始化同步

DataInitializer 已正确设置岗位的 parent 和 grade，无需修改。

### 3.6 Mock 脚本同步

generate-mock.sh 增加职级创建步骤，岗位创建请求增加 gradeId 参数。

- 前端：attendance.js、employees.js、api.js
