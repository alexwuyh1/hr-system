# 迭代开发缺陷分析

## 问题描述

自动登录验证请求了错误路径 `/api/dashboard`，但实际接口是 `/api/dashboard/summary`，导致刷新页面时 token 验证失败，自动退出登录。

## 根因分析

### 直接原因

[app.js:L790](file:///home/alex/hr-system/src/main/resources/static/app.js#L790) 硬编码了错误路径：

```javascript
// 错误
const response = await fetch(`${API_BASE}/dashboard`, {

// 正确
const response = await fetch(`${API_BASE}/dashboard/summary`, {
```

### 深层原因

这个错误在迭代过程中出现，说明开发存在以下缺陷：

**1. 缺少 API 契约管理**
- 前后端接口路径没有统一的契约文档或类型定义
- 前端代码中 API 路径散落各处，没有集中管理
- 修改后端路径时，前端没有同步更新

**2. 缺少自动化测试**
- 没有端到端测试覆盖自动登录流程
- 没有接口连通性测试
- 错误只能在手动测试时发现

**3. 代码修改缺乏影响分析**
- 修改 `DashboardController` 路径时，没有检查前端引用
- 没有使用 IDE 的全局搜索功能验证影响范围
- 修改是孤立的，没有考虑上下游依赖

**4. 前端代码组织不规范**
- API 路径硬编码在业务逻辑中
- 没有使用常量或配置集中管理
- 同样的问题之前也出现在数据导出路径上（`/api/data/export` 重复前缀）

**5. 缺少开发检查清单**
- 修改 API 路径后，没有验证前端调用
- 没有运行完整的冒烟测试
- 依赖用户手动发现问题

## 改进建议

| 缺陷 | 改进措施 | 优先级 |
|------|---------|--------|
| API 路径散落 | 前端集中定义 API 常量 | P0 |
| 缺少测试 | 添加基础冒烟测试 | P1 |
| 影响分析缺失 | 修改 API 前全局搜索引用 | P0 |
| 开发流程 | 建立修改检查清单 | P1 |

## 与 MVP 架构策略的关联

这正好验证了 [MVP架构与迭代策略.md](file:///home/alex/hr-system/doc/MVP架构与迭代策略.md) 中提到的原则：

- **单一入口点**：API 路径应该集中管理，而不是散落各处
- **代码约束优于文档**：用代码结构保证一致性，而不是依赖记忆
- **小步验证**：每次修改后立即验证上下游功能
